<objective>
Create the vllm-smollm2-disk Deployment (Pattern B — disk-based model loading) with initContainer
download, emptyDir sizeLimit, sha256 verification, and /model/READY sentinel; plus the NodePort
30203 Service. Deploy and verify the full Pattern B flow, including observing the emptyDir
re-download teaching moment.

Purpose: This is the core Pattern B deliverable. The initContainer downloads the model from MinIO
at pod startup rather than pulling it as an OCI image layer (Pattern A). The emptyDir volume means
the model is re-downloaded on every pod restart — which is the deliberate pedagogical trade-off vs
OCI ImageVolume.

Output: solution/ and starter/ manifests for the Deployment and Service; vllm-smollm2-disk running
and serving at localhost:30203; re-download behavior observed.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| initContainer → MinIO ClusterIP | mc downloads model via http://minio.minio:9000 — unencrypted in-cluster |
| host → vLLM NodePort 30203 | Student curl → vLLM inference — unauthenticated OpenAI-compat API |
| emptyDir /model | Shared volume between initContainer and vLLM main container — sentinel-gated access |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-05 | Tampering | emptyDir /model shared between containers | mitigate | Sentinel file /model/READY prevents main container from reading partial downloads; sha256 verification confirms integrity before sentinel write |
| T-03-06 | Information Disclosure | minio/minio123 in Deployment spec env | accept | Course-grade credentials; local dev cluster only; lab text explicitly notes demo-grade |
| T-03-07 | Denial of Service | emptyDir sizeLimit eviction | mitigate | sizeLimit: 1Gi set with 2× headroom for 517 MB model; ephemeral-storage requests/limits set on both containers per Research Pitfall 2 |
| T-03-SC | Tampering | quay.io/minio/mc image in initContainer | mitigate | Package Legitimacy Audit in RESEARCH.md confirms official MinIO org image; pinned to exact dated tag |
</threat_model>
