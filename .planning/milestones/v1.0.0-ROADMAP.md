## Overview

Rebuilt 302-llmops as a focused, comprehensive LLMOps course. Split AgentOps content to a companion course (303-agentops). Restored and modernized the original lab spine (Labs 00-09). All content runs CPU-only on a 16 GB KIND cluster.

## Milestone Summary

### Key Decisions

- **D-01 AgentOps split**: 302-llmops → LLMOps only; 303-agentops → AgentOps. Allows each course to go deep.
- **D-02 Three serving patterns**: Plain Deployment (Pattern A), KServe RawDeployment (Pattern B), vLLM Router (Pattern C). Side-by-side decision lab closes Phase 05.
- **D-03 Two packaging patterns**: OCI ImageVolume vs disk-based (MinIO initContainer). Decision tree in Phase 03.
- **D-04 No eval gate in Argo Workflows**: Eval belongs in 303-agentops. Pipeline lab teaches orchestration only (data→index→train→merge).
- **D-05 GPU as instructor demos**: All student hands-on labs CPU-only (16 GB laptops). GPU content deferred to v1.1 using instructor's GCP credits.
- **D-06 KServe RawDeployment (no Knative)**: Avoids 1.5 GB Knative+Istio overhead. Separate NodePort Service required (KServe reconciles managed Service to ClusterIP).
- **D-07 gitops/ inside course-code/labs/lab-11/**: ArgoCD repoURL must be HTTPS GitHub fork — file:// can't reach host filesystem from in-cluster pod.
- **D-08 ArgoCD probe timeout 15s**: Default 1s timeouts cause continuous restarts on KIND; 5s insufficient; 15s / failureThreshold 5 stable.
- **D-09 gitops/model-version in spec.template.metadata.annotations**: Must be pod template annotation (not Deployment metadata) to trigger rolling restart.
- **D-10 KEDA serverAddress**: `kps-kube-prometheus-stack-prometheus.monitoring.svc.cluster.local:9090` (release=kps prefix determines FQDN).
- **D-11 E2E loop fully automated**: alpine/git + SSH deploy key in Argo Workflows promote step enables single-push trigger for entire LLMOps chain.
- **D-12 vllm:num_requests_waiting**: Colon prefix (not underscore) confirmed for vLLM 0.9.1.

### Issues Resolved

- KIND ArgoCD crash-loop: liveness probe timeout 1s too short for `healthz?full=true` → patched to 15s
- KServe predictor Service: RawDeployment controller reconciles back to ClusterIP → separate NodePort Service required
- ArgoCD Application paths: resolved from `labs/lab-11/gitops/` → `labs/lab-11/solution/gitops/`
- KEDA admission for KServe Pattern C: requires `autoscalerClass=external` annotation before ScaledObject
- Argo Workflows CRD install timeout: pre-install Job times out → `--skip-crds` workaround documented
